Part 1 • 第 1 章

認識 RPA 與 PAD

🤖

什麼是 RPA (機器人流程自動化)？

想像你是一名每天要做 100 次漢堡的員工，動作一成不變：放麵包 ➡️ 煎肉排 ➡️ 放生菜 ➡️ 放起司 ➡️ 包裝。這是不是一件枯燥、容易累、且毫無成長性的工作呢？

RPA (Robotic Process Automation) 就是利用軟體機器人，去模擬人操作電腦的軌跡（例如開 Excel、複製貼上、點擊網頁、發 Email），自動幫你完成一模一樣的工作。

重要觀念！ RPA 自動化的是「電腦操作與點擊」，並非實體會走路的鋼鐵機器人！

每日重複的工作流程對比

人工處理
  • 開啟 Excel 逐筆複製訂單
  • 手動開啟瀏覽器登入後台
  • 肉眼比對輸入資料並按儲存
  • 複製發信模板，發送 Email 報表
RPA 自動

一個執行鍵 (Click)，上述複雜動作由軟體在 5 秒內無誤差、一氣呵成自動完成！

什麼是 Power Automate Desktop (PAD)？

PAD 是微軟推出的主打 RPA 工具。你不需要寫大量複雜的 Code，只需要用像「拖拉積木」一樣的方式拖曳預設的「Action（動作）」，即可建立出高度彈性的自動化流程。

❶ 動作 (Actions) 區 比喻：漢堡店的工具材料箱 內含檔案管理、Excel、網頁爬蟲、Email 發送、訊息方塊等各種現成模組。
❷ 流程設計 (Flow) 區 比喻：漢堡組裝配方步驟 由上而下排列被拖拉進來的 Action，PAD 將會依序乖乖執行。
❸ 變數 (Variables) 區 比喻：食材盛裝與傳遞盆 用來暫時存儲中間運算結果（例如從網頁抓到的訂單號），傳給下一個 Action 使用。
Part 1 • 第 2 章

認識流程 (Flow)

🔄

流程 (Flow) 就是為了完成某件特定的事，而將一連串動作 (Action) 按正確的順序排放。

正確的做漢堡流程 (依序由上而下執行)

① 烤麵包
② 煎肉排
③ 放生菜起司
④ 包裝出餐

錯誤的混亂流程 (當順序顛倒時...)

① 先組裝並包裝
② 再丟去煎肉排
③ 最後去烤麵包
PAD 執行法則： PAD 會 100% 聽從指令，在執行時由上而下，一列一列按部就班執行，不會主動幫你修正錯誤順序。因此在規劃流程時，順序的邏輯性是工程師的首要功課！
Part 1 • 第 3 章

認識變數 (Variable)

📦

變數就像是一個「有標籤的密封容器」，用來存放前面步驟產生的 Output (產物)，讓後面的步驟（Actions）能夠取出來用。

① Input (材料) 「麵包 + 奶油」
② Action (動作) 執行：「烤麵包」
③ Output (結果) 熱騰騰「烤好的麵包」
④ Variable (容器) %烤好的麵包%

📝 PAD 中的變數語法 `%變數%`

微軟的 PAD 主要是使用百分比符號 % 來辨識這是一個變數。例如輸入 `%烤好的麵包%`，PAD 就會知道這不是一般文字，而是要去名叫「烤好的麵包」的箱子裡撈東西。

預設命名 (不推薦) %NewVar%, %Text%, %Result%
缺點：當步驟有上百步時，根本不知道 NewVar 裡裝的是烤肉排還是冰可樂。
見名知意命名 (強烈推薦) %煎好的肉排%, %客戶Email%, %訂單金額%
優點：流程清晰明瞭、好維護，後面的人一秒就能看懂你的設計邏輯！

變數標記與數值運算

📐

▼ 變數的標記方式

在 Power Automate Desktop 中，為了精準存取各種資料型態（特別是包含行列結構的 Excel 資料表），% 符號的使用有著嚴格的語法規則：

1. 變數的左右必須以 % 符號括住
〔例〕 %Var_Name%
2. 如要加上文字或其他單位標註，需將文字放置在 % 符號之外
〔例〕 %Var_Number% 公斤。
資料表（DataTable）與資料列 / 資料行標記法：

3. 若變數內容為資料表，需要取出該資料表的特定資料列，「列號碼」必須放置在 % 符號內：

%ExcelData[2]%

4. 若變數內容為資料表，需要取出該資料表的特定資料行，「欄位名稱」必須放置在 % 符號內：

%ExcelData['姓名']%

5. 若變數內容為資料表，需要取出該資料表的特定欄位，「列號碼」與「欄位名稱」必須放置在 % 符號內：

%ExcelData[2]['姓名']%

請點選下方表格中的任何一列，或更換欄位，親眼看見 PAD 對應的進階取值語法變化：

1 李大同 NT$ 52,000

輸出語法：%ExcelData[2]['姓名']%

▼ 數值運算

在 Power Automate Desktop 中可使用基本算術運算子計算數值，並且可如同數學計算般以括號設定變數運算的優先順序：

+ 加法 (+) - 減法 (-) * 乘法 (*) / 除法 (/)
數值運算範例與核心法則

若要將變數與數值進行運算，運算符必須放置在 % 符號內：

〔例〕 %Var_Number + 120%

若要將多個變數進行運算，運算符必須放置在 % 符號內，且變數之間不需要再加上 % 符號：

〔例〕 %Var_A + Var_B - 100%

認識條件判斷 (Condition)

🌿

在做漢堡時，客人可能只點「單點」或加點「超值套餐」，兩者的製程不同。條件判斷 (Condition) 就是讓 PAD 能夠靈活變通，根據不同情況決定要執行哪一條流程！

🍔 漢堡店的條件決策樹

顧客點餐輸入 取得 %訂單內容%
❓ %訂單種類% 是「套餐」嗎？
是 (YES)
🍟 執行 Action：炸薯條與倒可樂
否 (NO)
🍔 執行 Action：直接漢堡裝袋出餐

利用 IF / ELSE 語法，每次執行只會挑選「符合條件的那一條分支」走，走完之後再匯合到主流程，完成出餐。這樣才不會點單點的客人，也拿到了不要的薯條或飲料。

Part 1 • 第 5 章

認識迴圈 (Loop)

🔁

今天，漢堡店一口氣湧進了 100 張訂單！難道身為店長的你，要為每一位客人都複製貼上一套全新的「做漢堡 100 步流程」嗎？

如果不使用 Loop (迴圈)

  • • 必須在 PAD 中複製相同流程 100 次。
  • • 流程圖變得極端冗長（有上千行動作）。
  • • 如果想要修改其中一個配方，得修改 100 次，保證改到頭昏腦脹、錯誤百出！

如果使用 Loop (迴圈)

只需要寫好 1 組漢堡流程，然後指示 PAD：

「重複執行直到沒有下一張訂單為止」

無論是 100 張、1,000 張甚至 1 萬張單，PAD 都會自自動動不知疲倦地循環完成！

🔁 PAD 迴圈邏輯圖
📥 取得今日第 1 張單 🍔 製作 1 份漢堡 (主流程) ❓ 還有下一張單嗎？
是 (回到主流程繼續做) 否 (結束出餐)

動作 (Action) 概念篇

🔧

在 Part 1 中，我們學習了自動化的哲學（流程、變數、條件、迴圈）。現在，我們將學習 PAD 最強力的內在驅動：動作 (Action)。

💡 Action 就像廚師的工具箱

炒菜需要平底鍋，裝盤需要白餐盤。處理自動化工作時也是如此。

PAD 預置了幾百種常用工具。想要處理 Excel 資料，就拉出 Excel 動作組；需要串接 API 或網頁，就拉出 HTTP / 瀏覽器動作組。

💡 核心觀念： 學會了使用工具箱，就等於完全征服了 PAD 自動化流程！

🏆 自動化發展的 4 步驟流程導向

1. 取得資料 Excel、CSV、網頁資料、Email 附加檔案
2. 處理資料 字串處理 (Text)、日期時間、資料表 (DataTable)
3. 執行流程 條件判斷 (Condition)、迴圈控制 (Loop)
4. 輸出結果 寫入 Excel、匯出 PDF 報表、發信通知進度

取得資料 (Input)

📥

要開始新的一天工作，店長得先確認今天有哪些訂單。如果沒有取得輸入 (Input)，機器人根本不知道今天要幹嘛。訂單常常放在 Excel 或 CSV 中。

PAD 常用的 5 大 Excel Actions

1. 啟動 Excel

開啟指定的 Excel 活頁簿檔案路徑。

2. 設定工作表

指定要撈資料的 Sheet (例如：今日訂單)。

3. 取得可用行列

自動偵測表格大小邊界，避免少讀或多讀。

4. 讀取 Excel

將儲存格中訂單資料全部取出放到變數中。

5. 關閉 Excel

可選取存檔關閉或不存檔直接關閉。

📋 Excel 範例：今日訂單.xlsx

Part 2 • 單元 2

處理資料 (Process)

⚙️

取得訂單資料後，常需要先進行資料的「整理與淨化」才能進入製作（比如縮寫轉換「牛堡」轉換為「牛肉漢堡」）。PAD 提供了 4 個核心資料處理模組：

Text (文字動作)

取代文字、分割文字、擷取部分文字

比喻： 幫漢堡縮寫正名。把 Excel 裡雜亂的「雙層牛」取代並正規化為標準的「雙層牛肉堡」。

DataTable (資料表)

管理表格列與欄、篩選、新增資料

比喻： PAD 的整張店內點餐大看板。用來載入整張訂單 Excel 的完整行列結構，方便隨意取用某欄位。

DateTime (日期時間)

取得目前日期、格式化時間、增加時間

比喻： 紀錄漢堡出餐時間。產生格式化檔名「今日訂單_20260712.xlsx」來方便歸檔。

List (清單)

建立新清單、新增項目、清單排序

比喻： 客人今日點的配料清單：[起司, 生菜, 番茄, 培根]，用來方便後面迴圈一一查驗。

執行流程 (Execute)

🚀

整理好資料，開始動手做漢堡！這時候會結合 Condition (條件判斷) 動作。PAD 有各種不同的判斷，我們來比較一下：

條件分支語法 適用時機 漢堡店經典範例
IF 符合條件才做，不符合就直接略過 IF 是套餐 ➡️ 炸份薯條給他（單點的話直接跳過此步）。
IF / ELSE 二選一必走其一，都有對應要做的事 IF 內用 ➡️ 擺在塑膠托盤；ELSE ➡️ 塞入外帶紙袋。
ELSE IF 多選一，有多種情況各做不同的事 IF 飲料是可樂 ➡️ 拿可樂杯；ELSE IF 紅茶 ➡️ 拿紅茶杯；ELSE ➡️ 拿熱咖啡杯。
💡 黃金拍檔：Loop + Condition !
最常見的進階流程是：先用 For Each 迴圈 一筆一筆巡迴撈取 Excel 的訂單列。在迴圈內部嵌入一個 If 條件判斷 判斷是否為套餐，再各別出餐。兩者完美整合，能完成 99% 的複雜商務流程！
Part 2 • 進階單元

For Each 迴圈的奧秘

🔁

我們在 Part 1 認識了基本迴圈。在實戰動作中，最常用的就是 For Each 迴圈。它是用來自動巡迴處理整張表格 (DataTable) 或清單 (List) 的黃金工具！

漢堡店的「排隊點餐」比喻

櫃台前排了 5 位客人（這是一張 DataTable / List 容器）。 身為店員，你不會一次做 5 個漢堡，而是「一個接一個」處理：

  • 叫第一位客人 ➡️ 製作漢堡並出餐
  • 叫第二位客人 ➡️ 製作漢堡並出餐
  • 一直重複，直到隊伍中沒有下一位客人為止！

核心變數：%CurrentItem%

在 PAD 中拖入 For Each 動作時，它會要求你指定要讀取的變數（例如 %今日訂單%），並自動產生一個名為 %CurrentItem%（目前項目）的迭代容器。

// 在每一次的循環中： 第一次：%CurrentItem% = 第 1 張訂單內容
第二次：%CurrentItem% = 第 2 張訂單內容...

📊 For Each 是如何分割 DataTable 的？

整張 Excel 訂單 %今日訂單% (DataTable)
列 1: 牛肉漢堡, 數量 2
列 2: 雞腿堡, 數量 1
列 3: 可樂, 數量 3
For Each 迴圈內部 每次只掏出「一列」放進：
%CurrentItem%
= [牛肉漢堡, 數量 2]
實戰小技巧：如果想讀取目前項目的特定欄位，可以使用 %CurrentItem['欄位名稱']% 或索引值 %CurrentItem[1]%！例如：%CurrentItem['數量']% 就會撈出目前客人的點餐數量！

輸出結果 (Output)

📤

漢堡做完了，必須交到客人手中。在商務自動化中，輸出結果 (Output) 是將流程算好的重要資料存檔或傳送出去，通常有以下 3 種最熱門的輸出方式：

1. 回填 Excel 資料表

將訂單狀態更新為「已完成」，並填入回報的出餐時間。讓人員開啟 Excel 就能一目瞭然。

2. 匯出 PDF 銷退貨收據

自動產生美觀不可修改的 PDF 出餐明細，直接發給外送司機當作收據憑證。

3. 發送 Email 通知信

調用 Outlook 動作組，自動向主管發信匯報「今日 100 筆訂單自動化包裝漢堡已全數完成！」
